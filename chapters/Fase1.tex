\chapter{Fase 1: Planificación y Diseño}

\begin{abstract}
Este capítulo recoge el trabajo realizado durante la Fase 1 del proyecto, centrada en la
planificación inicial y en el diseño de la base técnica del sistema.
\end{abstract}

\section{Objetivos de la fase}
Durante esta primera fase se establecieron las bases organizativas y documentales del proyecto.
El objetivo principal fue dejar preparado todo el entorno de trabajo necesario para poder avanzar
con seguridad en las fases posteriores, tanto en el desarrollo del videojuego como en la redacción
de la memoria.

En esta etapa se definió la estructura inicial del repositorio, se organizó el sistema de control de
versiones y se preparó la documentación base que serviría como referencia para el resto del trabajo.

Asimismo, se buscó delimitar con claridad el alcance inicial del proyecto, dejando fijadas las piezas
fundamentales que iban a guiar todo el desarrollo posterior. Era importante que desde el comienzo
existiera una base común para tomar decisiones, registrar avances y mantener alineadas la parte técnica
y la parte documental.

\section{Trabajo realizado}
Durante esta fase se instanció y configuró todo lo necesario para trabajar con el proyecto de forma
ordenada. Por un lado, se preparó el repositorio del proyecto en GitHub como sistema de control de
versiones, de forma que todo el desarrollo pudiera quedar registrado y organizado desde el inicio.

Por otro lado, se creó y estructuró el espacio destinado a la memoria, de manera que ambos elementos
quedaran conectados dentro de un mismo flujo de trabajo. Esto permitió que el avance técnico y la
redacción del documento fueran de la mano desde el principio.

Además, se recopilaron y definieron los principales requisitos del proyecto, a partir de los cuales
se elaboraron varios documentos base: el GDD, el TDD, la planificación y gestión del proyecto y el
apartado de pruebas y resultados. Estos documentos sirvieron como plantilla y referencia para empezar
la memoria con una estructura clara y coherente.

El GDD permitió fijar la idea global del videojuego, concretando la propuesta jugable, el contexto
narrativo y las mecánicas principales que debían aparecer en la memoria. El TDD trasladó esa idea a
un plano más técnico, ayudando a organizar la arquitectura inicial y a preparar la base sobre la que
se construirían los sistemas del juego.

La planificación y gestión del proyecto recogió la división temporal del trabajo, el orden de las
fases y la forma de abordar cada bloque de tareas, mientras que el documento de pruebas y resultados
se planteó desde el principio como el lugar donde se iría dejando constancia de la verificación del
trabajo realizado y de los resultados obtenidos durante el desarrollo.

En conjunto, esta fase no solo preparó el entorno de trabajo, sino que también dio forma al marco de
referencia con el que se escribiría la memoria. Gracias a ello, el documento pudo comenzar con una
estructura coherente, apoyada en información ya organizada y lista para ser desarrollada en las fases
posteriores.

Una vez que todo quedó registrado, organizado y contemplado en la memoria, se consideró finalizada
esta fase y se dio paso a la siguiente etapa del proyecto.

\section{Entregables de la fase}
Los principales entregables de esta fase fueron la configuración inicial del repositorio, la base
organizativa del proyecto y la documentación de partida necesaria para comenzar el desarrollo y la
redacción de la memoria.

También quedaron definidos los documentos de soporte que se utilizarían como referencia en las fases
posteriores, especialmente para orientar la estructura de la memoria y la planificación del trabajo.

Entre los entregables más importantes estuvo la creación de una base documental suficientemente sólida
como para permitir que el resto de capítulos se construyeran sobre criterios comunes. Esto evitó que
la memoria naciera como una colección de textos aislados y permitió que cada apartado se relacionara
con el resto del proyecto desde el principio.

\section{Incidencias y ajustes}
Durante esta fase no se trabajó todavía sobre la parte funcional del videojuego, por lo que los ajustes
se centraron principalmente en la organización del proyecto, la preparación de la documentación y la
alineación entre la estructura del repositorio y la memoria.

El resultado fue un punto de partida estable, con la información necesaria ya recopilada y una base de
trabajo adecuada para continuar con la implementación del núcleo del proyecto.
